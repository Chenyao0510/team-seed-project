#### Screen 2: Integration Map (視座の獲得レポート画面)

議論を終え、得られた視座（構造）を可視化する結論画面。
この画面の目的は「結論を提示すること」ではなく、
議論によって見えてきた構造と、ユーザーの介入が与えた影響を可視化することである。

* **Growth Header:**
    * 小さく Before / After を表示する。
    * Before / After は主役ではなく、構造マップを読むための導入として扱う。

    ```
    Before:
    ハッカソンは面白いのだろうか？

    ↓

    After:
    ハッカソンはどのような成果や価値を生み出すのだろうか？
    ```

* **Structure Map (Center / Main Hero):**
    * この画面の主役。
    * Bento UI を用いて、議論から抽出された主要構造を可視化する。
    * 単なる箇条書きではなく、「要素同士の関係性」を表現することを重視する。
    * カード間は線・発光・アニメーション等で接続し、
      「どの概念がどの概念へ繋がっているか」を直感的に理解できるようにする。

    例:

    ```
                [個人成長]
                     │
                     │
    [コミュニティ] ─ [ハッカソン] ─ [成果創出]
                     │
                     │
                [社会実装]
    ```

    * 各カード内部に具体要素を配置する。

    例:

    ```
    個人成長
    ├ 技術習得
    ├ 学習効率
    └ 挑戦経験

    成果創出
    ├ 新規アイデア
    ├ プロトタイプ
    └ 事業可能性
    ```

* **User Intervention Trace (重要):**
    * ユーザーが追加した観点・質問・異議を、
      マップ内のどの構造へ影響したか可視化する。
    * 単なる文章ではなく、構造上で直接示す。

    例:

    ```
    ★ あなたの介入
    「実用性は？」

           ↓

        [成果創出]
           ↓
        [社会実装]
    ```

    * Framer Motion を用いて、
      ユーザー介入 → 構造変化
      が順番に発光しながら表示される。

* **Animation:**
    * ランダム表示は禁止。
    * 構造が組み上がっていく演出を採用する。
    * 表示順:

    ```
    Before/After
      ↓
    中心テーマ
      ↓
    構造カード
      ↓
    関係線
      ↓
    ユーザー介入の影響
    ```

    * 「議論の結果を見る」ではなく、
      「構造が構築される様子を見る」体験を目指す。

* **Feedback Area:**
    * 長文説明は禁止。
    * 1〜2行でユーザーの影響のみ伝える。

    例:

    ```
    あなたの「実用性は？」という介入により、
    議論は「面白さ」から「成果創出」へ拡張されました。
    ```

    または

    ```
    あなたの視点は、
    この議論の中で「社会実装」という構造として統合されました。
    ```

---

#### UX方針

* Screen 2 は「結論画面」ではない。
* Screen 2 は「議論構造の可視化画面」である。
* 主役は Before/After ではなく Structure Map。
* 主役は文章ではなく関係性。
* 主役は要約ではなく構造。
* ユーザーは「答えを得た」のではなく、
  「どのような構造で考えるべきかを持ち帰る」体験を得る。
