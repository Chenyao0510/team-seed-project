# AI × Growth Hackathon: "Insight Navigator" Project Context

## 1. Project Overview
本プロジェクトは、「AIに答えを代行させる」のではなく「AI同士の議論に人間が介入し、複雑な事象、問いの『構造』を理解して視座を高める（Growth）」ためのWebアプリケーションである。

**Core Principle:** 答えを出すのではなく、「構造」と「問いの進化」をユーザーに持ち帰らせる。

---

## 2. UI/UX Specifications
アプリケーションは主に3つの画面（Setup → Stage → Map）で構成される。認知負荷を下げるため、ギャルゲ（ビジュアルノベル）のようなキャラクター横並びのレイアウトと、流麗なFramer Motionのアニメーションを前提とする。

### Screen 0: Setup (テーマ入力＆初期メンバー編成画面)
議論のテーマと、最初に参加させるAIキャラクターを決定するエントリー画面。
* **Input:** 議論したいテーマを入力。
* **Member Selection:** 初期メンバーの名前を入力/選択する。
* **Avatar Pipeline:** ここで指定された初期メンバーに対しても、バックグラウンドで「動的アバター生成パイプライン（Gemini検索→Nanobanana生成→透過）」を実行し、アバター画像を準備してからScreen 1へ遷移する。

### Screen 1: Debate Stage (討論＆介入画面)
AI同士の議論を「観察」し、ユーザーがリアルタイムに「介入」するメイン画面。チャットUIではなく「ステージ演出」を用いる。

* **Header:**
    * Center: 議論のテーマ（不変）
    * Left: 現在フォーカスされているトピック
    * Right: 「過去のチャットログ閲覧」ボタン（押下でスライドイン）
* **Stage (Center):**
    * 参加キャラクター（AI複数体 ＋ User）のアバターが横並び。ユーザーのアバターは**一番右に固定**。
    * 発言中、または思考中のキャラクターのみがハイライト（拡大・発光）される。
    * 発言内容は、各キャラの吹き出しではなく、画面中央下部の「固定の大きなテロップエリア（ギャルゲのテキストボックス風）」に表示される。
* **Side Panel (Left):**
    * 現在の議論構造を表すミニマップ。
    * Geminiが抽出した主要論点（3〜6個程度）をカードとして表示する。
    * 新たな論点が出現した場合はアニメーション付きで追加される。
    * ユーザーが全ログを追わなくても「今どの論点について話しているのか」が把握できることを優先する。
* **History Modal/Drawer (Right):**
    * 過去の会話履歴を表示するUI。LINEのグループチャット風のデザイン。
    * 各発言の横に、キャラクターのアバター画像を丸枠で小さくアイコン化して配置する。
* **Action Bar (Bottom):**
    * ユーザー介入ボタン群：「人物追加」「異議を唱える」「観点追加」「質問」。
    * **「人物追加」のアクション:** 人物選択モーダルが開き名前を入力する。Screen 0と同様の「動的アバター生成パイプライン」を経て、ステージ上に新規キャラクターのアバターが追加される。
    * **「異議・観点・質問」のアクション:** 選択すると入力モードになる。キーボードで入力した内容は、キャラのセリフと同様に中央のテロップエリアに表示され、AIへ送信される。
    * 終了ボタン：「議論を整理する」（※APIを叩いてScreen 2へ遷移）。
* **Reflection Turn:**
    * 一定ターンごとに討論を一時停止する。
    * これは「介入の強制」ではなく、人間へ思考の主導権を返すための区切りである。
    * AIは次に考えるべき論点や追加人物を具体的には提案しない。参加者以外のAI(facilitator)が現状の議論を要約して行動を起こすような一言をユーザに対して発する。
    * 現在の問いと論点構造のみを表示し、ユーザーが以下を選択する。
        * 続きを見る
        * 人物追加
        * 観点追加
        * 異議を唱える
        * 議論を整理する(終了する)

### Screen 2: Integration Map (視座の獲得レポート画面)
議論を終え、得られた視座（構造）を可視化する結論画面。二項対立ではなく「統合された構造」を見せる。

* **Growth Header:**
    * 「彼らとの議論を経て、問いはこのように再定義されました」
    * `[Before 問い]` ➔ `[After 問い]`
      （議論によってテーマがどのような構造として捉え直されたかを示す）

* **Map Area (Center):**
    * 枠や境界線を排除した、クリーンなBento UIスタイル。数個の大きなカード（レイヤー・カテゴリ）があり、その中に具体的な要素が階層的に配置される。
    * **Animation:** ランダム表示ではなく、Framer Motionを用いて要素が順次組み上がっていく（stagger）「構造構築」のアニメーションを実装する。
    * ユーザーがScreen 1で追加した「観点」が、マップ内のどの要素になったかハイライト（★マーク等）される。
    * ※この画面の主役は「After問い」ではなく「構造マップ」である。After問いはマップを読むための入口として扱う。

* **Feedback Footer:**
    * 「あなたが追加した『[ユーザーの介入内容]』は、この議論の中で『[マップ上の要素]』として取り込まれました」
    * または
    * 「あなたの『[ユーザーの介入内容]』という視点が、新たな論点として議論に加わりました」
